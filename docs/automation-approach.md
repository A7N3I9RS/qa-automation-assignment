# Automation Approach

## Architecture

The project uses Playwright Test with TypeScript.

UI tests use page objects from [src/pages](../src/pages) to keep selectors and page-level actions away from test assertions. Shared scenario-level behavior is implemented in [src/scenarios](../src/scenarios).

API tests use Playwright's built-in `request` fixture and external test data from [tests/api/data](../tests/api/data).

## Why This Coverage Was Prioritized

The first automation priority is the main business flow:

```text
User buys a product
```

That flow gives the highest regression value because it crosses authentication, product selection, cart review, checkout information and order completion.

After the happy path, the suite adds targeted risk coverage:

- Locked-out login behavior
- Catalog sorting
- Cart state changes
- SauceDemo special-user differences
- API data and schema checks
- API create-user response-time check

## Handling Known Defects

SauceDemo intentionally exposes defects through dedicated users such as `problem_user`, `performance_glitch_user`, `error_user` and `visual_user`.

The special-user matrix keeps these failures visible. This makes the report useful for triage: `standard_user` shows the expected baseline, while failing users identify which shared shopper behavior is currently broken.

## Test Commands

| Command | Purpose |
| --- | --- |
| `npm test` | Run the full Playwright suite |
| `npm run test:baseline` | Run stable baseline UI coverage |
| `npm run test:ui` | Run all UI tests |
| `npm run test:ui:headed` | Run UI tests headed |
| `npm run test:matrix` | Run SauceDemo special-user matrix |
| `npm run test:matrix:headed` | Run SauceDemo special-user matrix headed |
| `npm run test:defects` | Alias for the defect-focused matrix |
| `npm run test:defects:headed` | Run defect-focused matrix headed |
| `npm run test:api` | Run API tests |
| `npm run report` | Open the Playwright HTML report |

## Reporting and Debugging

Playwright is configured to retain useful artifacts on failure:

- Trace: retained on failure
- Screenshot: only on failure
- Video: retained on failure
- HTML report: generated by Playwright

## CI

The repository includes a GitHub Actions workflow at [.github/workflows/playwright.yml](../.github/workflows/playwright.yml).

To run ReqRes API tests in CI, configure a repository secret named `REQRES_API_KEY`.
